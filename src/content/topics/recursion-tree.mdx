Each recursive call spawns new calls, forming a **tree**. The shape of that tree — how many branches and how deep — determines the complexity.

## Key Properties

- **Branching factor** — how many children each node spawns
- **Depth** — how deep the recursion goes  
- **Work per level** — cost at each level (often O(n))
- **Total work** — work per level × number of levels

## Complexity Patterns

| Function | Branches | Depth | Work/Level | Total |
|----------|----------|-------|------------|-------|
| Factorial | 1 | n | O(1) | O(n) |
| Fibonacci | 2 | n | grows | O(2^n) |
| Binary Search | 1 | log n | O(1) | O(log n) |
| Merge Sort | 2 | log n | O(n) | **O(n log n)** |

**Why is Merge Sort O(n log n)?** Each level does O(n) work (merging), and there are log n levels. Branching × depth gives the exponent, but **work per level** brings it down.

## Step-by-Step Examples

### Factorial — 1 Branch

```
factorial(4)
└── factorial(3)
    └── factorial(2)
        └── factorial(1)
            └── factorial(0) → 1
```
Each call spawns exactly one child. Total calls = depth. Complexity = O(n).

### Fibonacci — 2 Branches

```
fib(4)
├── fib(3)
│   ├── fib(2) → fib(1)
│   └── fib(1)
└── fib(2) → fib(1)
```
Same subproblems solved repeatedly. That's why naive Fibonacci is O(2^n).

### Binary Search — 1 Branch, Halved

```
search(0..15)
└── search(0..7)
    └── search(0..3)
        └── search(1..1) → found
```
Each call halves the range. Complexity = O(log n).

## Common Mistakes

1. **Thinking branch count alone determines complexity** — Binary search has 1 branch but is O(log n) because range halves each level.

2. **Ignoring work per level** — Merge sort has 2 branches but is O(n log n) because each level does O(n) work.

3. **Confusing tree depth with call count** — A tree with n/2 nodes per level has more total work than a tree with 1 node per level.

4. **Forgetting memoization** — With memoization, Fibonacci eliminates overlapping subproblems and becomes O(n).

## Practice

1. **Trace**: Draw the recursion tree for `sumArray([1,2,3,4])` implemented as `sum(arr, n) = arr[n] + sum(arr, n-1)`

2. **Compare**: At depth 6, how many calls does Fibonacci make vs Factorial?

3. **Analyze**: Why does `pow(2, n)` via `pow(2, n-1) + pow(2, n-1)` give O(2^n)?

4. **Optimize**: How would memoization change Fibonacci's tree? Which nodes disappear?
